# 节奏 · Rhythm

一个**看重恢复、而不是完美**的学习节奏记录工具。

大多数学习计划工具都在优化"复习什么、何时复习"，并用"遵守率"衡量执行。但被生活打断之后，真正决定结果的，往往不是"漏没漏"，而是**漏了之后能不能回到正轨**。Rhythm 把这件被忽略的事放在中心：它记录你的作息、状态与学习安排，描述你最近的节奏，并据此给出更高容错的建议——而不给你贴任何"类型"标签。

🔗 **在线使用**：https://luancy0423.github.io/rhythm/

## 它记录与计算什么

- **作息**：上床/起床时间、工作日或休息日 → 睡眠规律指数（SRI）、社会时差
- **状态**：随时打卡的精力与心情（一天可多次）→ 日间与日内波动
- **学习**：每段计划是完成、被打断、还是补回 → 任务完成波动、遵守率
- **招牌指标 · 恢复动力学**：错过-恢复率（被打断后回到正轨的比例）、连锁率（一次错过是否引发接连错过）、恢复时长

所有指标都呈现为**会变动的描述**，不是诊断、不是固定标签。

## 隐私

- **本地优先**：你的记录默认只存在你自己的浏览器里（localStorage），不上传任何服务器。
- 可随时一键**导出 / 导入 JSON** 作为备份或迁移。
- 因为数据存在本地：请全程使用同一台设备、同一个浏览器，并定期导出备份。

## 设计原则

- 反焦虑、反标签：没有红色警告、没有"连胜"计数（那只会制造"一旦中断就崩"的压力）。
- 看重恢复：被打断会得到一个"做个最小版本、先接上链条"的温和提示，而不是指责。
- 循证：作息规律、间隔练习、按波动度留缓冲、if-then 预案等建议，均基于已有研究。

## 本地运行 / 二次开发

仓库里的 `index.html` 是自包含的单文件（React + recharts 已打包），直接用浏览器打开即可，无需任何依赖或构建步骤。

部署：把 `index.html` 放到任意静态托管（GitHub Pages / Netlify / Vercel）即可。本仓库已通过 GitHub Pages 发布于上面的链接。
